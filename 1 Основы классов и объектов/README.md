# Практика #1
Самостоятельно изучите принцип подстановки Лисков и приведите пример его использования.
# Ключевые принципы соблюдения LSP:
Не изменяйте поведение родительского класса — подклассы должны дополнять, а не изменять поведение.
Предусловия не должны быть усилены — подкласс не должен требовать больше, чем базовый класс.
Постусловия не должны быть ослаблены — подкласс должен гарантировать как минимум то же, что и базовый класс.
Инварианты должны сохраняться — подкласс должен сохранять все условия, которые всегда истинны для базового класса.
Используйте интерфейсы и абстрактные классы для определения общего поведения.
Избегайте наследования, если оно нарушает LSP — лучше использовать композицию.
