# Связи между классами:
# Внутри пакета animals:
Dog → Animal (наследование + тот же пакет)

Cat → Animal (просто использование + тот же пакет)

# Межпакетные связи:
ForeignAnimal → Animal (наследование + разные пакеты)

ForeignClass → Animal (просто использование + разные пакеты)

# Главный класс:
Main → все классы (тестирование извне пакетов)

# Цель структуры:
Демонстрация всех комбинаций доступов:

Один пакет, один класс - private доступ (Animal к своим private членам)

Один пакет, разные классы - default доступ (Cat к Animal)

Один пакет, наследование - protected + default доступ (Dog к Animal)

Разные пакеты, наследование - protected доступ (ForeignAnimal к Animal)

Разные пакеты, без наследования - public доступ (ForeignClass к Animal)

Недоступность - private и default из других пакетов

# Cценарии проверки:
Dog проверяет доступ к Animal (подкласс + тот же пакет)

Cat проверяет доступ к Animal (не подкласс + тот же пакет)

ForeignAnimal проверяет доступ к Animal (подкласс + другой пакет)

ForeignClass проверяет доступ к Animal (не подкласс + другой пакет)

Main проверяет доступ ко всем классам извне

Такая структура позволяет продемонстрировать ВСЕ возможные случаи применения всех четырех спецификаторов доступа в различных контекстах отношений между классами.
